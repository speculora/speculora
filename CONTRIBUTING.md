# Contributing

This repository is the public handbook for [Speculora](https://speculora.com) — the demand-and-saturation intelligence map for software opportunities. It explains the concepts behind the product: the task unit, the gap gradient, the opportunity score, and how to find and validate software opportunities.

Contributions that make this documentation clearer, more accurate, or more useful are welcome.

## What helps

- **Corrections** — fix errors, broken links, or unclear explanations.
- **Clarity** — tighten wording, improve examples, sharpen definitions.
- **New questions** — propose additions to the [FAQ](docs/faq.md) that real builders actually ask.
- **Glossary entries** — suggest terms that deserve a clean definition.

## How to contribute

1. Open an issue describing the change, or
2. Submit a pull request with a focused, single-purpose edit.

Keep the house style: authoritative and declarative, plain Markdown, clear headings, short paragraphs, consistent terminology (use the terms exactly as defined in the [glossary](docs/glossary.md)), and no hype. Cross-link related documents where it helps the reader.

## Scope

This repo is conceptual and reference documentation. It intentionally does not cover internal implementation. For the live data and the product itself, see [speculora.com](https://speculora.com).
